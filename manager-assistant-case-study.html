Situation
Providing feedback is
a responsibility.

When provided thoughtfully, feedback can guide an individual toward reaching their full potential. But when given superficially, it can do real harm — undermining trust and growth.

The client faced a complex challenge: how do you provide meaningful feedback at scale? For a single manager overseeing 10 people, feedback is already demanding. But for managers responsible for 200+ associates, the cognitive load becomes unmanageable.

A solution was taking shape as a 4-step AI-powered wizard — but something essential was still missing. That's where UX design came in.

"When done superficially, feedback can do more harm than good."
200+ Associates per manager needing structured, individual feedback
Version 1 · Old way of working
Manual, Excel-driven
The first iteration was built around existing workflows: managers had to download Excel reports, upload them into the tool, manually cross-reference data from multiple sources, and re-upload every time something changed. It worked — but it inherited every pain point of the process it was meant to replace.
  • Download → format → upload cycle, repeated for every review
  • Data went stale between each export and use
  • 5+ manual steps before any feedback could be drafted
→
Version 2 · AI-first rethink
Live database + AI-assisted
The redesign started from a different premise: what if the tool connected directly to the database, so data was always current? Paired with AI that could research, draft, and adapt tone automatically, the feedback process shifted from a manual chore into a guided, near-effortless flow.
  • Direct database connection — no uploads, ever
  • AI surfaces the right context per employee automatically
  • 2 steps from login to sending feedback
Task · Research & Reframe
The AI idea was
there. Nobody
listened yet.

Early in the project, a proposal to integrate AI and a live database connection was already on the table. It was quickly set aside — time constraints made it feel like a nice-to-have, not a necessity.

The research data existed. The problem was that no one was interpreting it. Numbers sat in spreadsheets; patterns went unnoticed; and the team moved forward building a tool that digitised the pain rather than removed it.

As the UX designer, I pushed back. Using the raw research data, I built an Empathy Map to make the user's reality impossible to ignore — translating numbers into human behaviour and emotion. That canvas became the turning point that reopened the AI conversation.

Task · Problems Defined
Three core problems
blocking adoption

With the Empathy Map making the user's reality concrete, the problems to solve became clear. My role was to mature the experience into something managers would genuinely want to use — not just a functional tool, but one that truly resembles a product.

Cognitive overload & fragmented data
Managers had to manually gather data from 3 separate tools, upload it, and hold it in memory — creating friction before they even started writing feedback.
→ Direct database connection
A feature hidden in plain sight
Users couldn't discover or leverage the AI research capabilities. Working in isolation, they missed insights that could unlock a richer, more data-backed feedback experience.
→ Surfaced AI as a core feature
Rough onboarding & accessibility gaps
The tool lacked language support and an accessible legal layer, limiting real-world adoption across diverse, international management teams.
→ Onboarding + language + legal
Action
Reducing friction,
surfacing intelligence

Based on data collected from users and the client, three features emerged as priorities. The goal was always the same: eliminate assumptions, reduce manual steps, and let AI do the heavy lifting — so managers could focus on quality, not logistics.

Live Database Connection
The manager gets a direct, pre-built connection to all relevant feedback data. No manual uploads, no copy-paste, no memory strain. The AI selects the right context for each associate automatically.
Eliminates the "last updated?" anxiety — data is always current and trusted.
AI Templates + Tone Customization
Three surfaced AI capabilities: context-rich templates (per employee), tone alignment (formal / coaching / direct), and language matching. Managers sync with the main persona once; the AI adapts.
Turns a blank page into a smart starting point — managers edit, they don't draft from scratch.
Onboarding, Language & Legal Layer
Improved onboarding reduced setup from 5 steps to 2. Language support opened the tool to international teams. A legal review layer was added as a final checkpoint before feedback is sent.
Reduces the barrier to first use and builds trust in the output before it reaches the associate.

Reflections
What this project taught
Hidden features are non-features
Powerful AI capabilities that users can't discover may as well not exist. Surfacing them intentionally — at the right moment — is core UX work, not optional polish.
Reduce cognitive load first
Before improving quality, remove friction. When managers spend less energy gathering data, they invest that energy in crafting genuinely useful feedback.
Measure what changes behavior
Completion rate — not just satisfaction scores — was the signal that mattered. Pairing qualitative UX research with behavioral metrics told the full story.
